# Structure Plan: UI Light Mode

## 1. Directory Structure Changes
The current project structure handles state largely inside `App.jsx`. To ensure modularity and clean separation of concerns, the theme management logic will be extracted into a custom React hook. 

We will introduce a new `hooks/` directory in `src/`.

### Added Files
- `src/hooks/useTheme.js`: A custom React hook responsible for:
  - Checking the initial OS theme preference (`window.matchMedia`).
  - Checking and persisting `localStorage` state.
  - Adding or removing the `.light-theme` class from `document.body`.
  - Exposing the current `theme` string and a `toggleTheme` function.

### Modified Files
- `src/index.css`: To refactor existing dark mode variables into semantic variables, define `.light-theme` overrides, and implement smooth CSS transitions.
- `src/App.jsx`: To consume the new `useTheme` hook and insert a toggle button (e.g., Sun/Moon icon) into the application header.

### `src/index.css` Modifications
```css
/* Update base variables for dark mode (default) */
:root {
  --bg-main: #0f111a;
  --bg-surface: #1e2130;
  /* ... refactored color names ... */
}

/* Define light mode overrides */
body.light-theme {
  --bg-main: #f8fafc;
  --bg-surface: #ffffff;
  /* ... corresponding light colors ... */
}

/* Add transition for smooth theme switching */
body, .panel, header, button, input {
  transition: background-color 0.3s ease, color 0.3s ease, border-color 0.3s ease, box-shadow 0.3s ease;
}
```

## 3. Structural Boundaries
- **UI Logic (`App.jsx`)**: Simply calls the `useTheme` hook, determines the correct icon to display, and renders the `<button>` in the header.
- **State/Side Effect Logic (`useTheme.js`)**: Encapsulates all interactions with `localStorage` and `document.body.classList`. The App component never touches `localStorage` or `classList` directly.
- **Styling (`index.css`)**: Defines the visual manifestation of the themes using semantic variables. No inline styling for theme colors.
